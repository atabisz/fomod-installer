<objective>
Enrich the `Instruction.UnsupportedFunctionalityWarning` with `reason` and `platform` fields so callers (Vortex) can build OS-specific user messages for C# script FOMADs on Linux.

Purpose: Callers currently receive only `type=unsupported, source=CSharpScript` with no actionable context. Adding `reason` and `platform` lets Vortex display "CSharpScript not supported on Linux" without hardcoding platform knowledge.

Output: Updated `Instruction` record with new properties, updated call site, updated TypeScript integration test.
</objective>
