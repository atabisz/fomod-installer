**New `build-ipc` job (matrix):**

```yaml
build-ipc:
  strategy:
    matrix:
      include:
        - os: windows-latest
          platform: win32-x64
        - os: ubuntu-22.04
          platform: linux-x64

  runs-on: ${{ matrix.os }}

  steps:
    - uses: actions/checkout@v6

    - uses: actions/setup-dotnet@v5
      with:
        dotnet-version: '9.x'

    - name: Install Node.js via Volta
      uses: volta-cli/action@v4

    - name: Setup pnpm
      uses: pnpm/action-setup@v4

    - name: Install dependencies
      run: pnpm install

    - name: Build C# IPC binary (Windows)
      if: matrix.platform == 'win32-x64'
      working-directory: src/ModInstaller.IPC.TypeScript
      run: node build.js build-csharp

    - name: Build C# IPC binary (Linux self-contained)
      if: matrix.platform == 'linux-x64'
      run: |
        dotnet publish src/ModInstaller.IPC \
          -c Release \
          -r linux-x64 \
          --self-contained true \
          -p:PublishSingleFile=true \
          -p:EnableCompressionInSingleFile=true \
          -o src/ModInstaller.IPC.TypeScript/dist/linux-x64

    - name: Set execute permissions (Linux only)
      if: matrix.platform == 'linux-x64'
      run: chmod +x src/ModInstaller.IPC.TypeScript/dist/linux-x64/ModInstallerIPC

    - name: Upload IPC binary artifact
      uses: actions/upload-artifact@v4
      with:
        name: ipc-${{ matrix.platform }}
        path: src/ModInstaller.IPC.TypeScript/dist/${{ matrix.platform == 'linux-x64' && 'linux-x64/' || 'ModInstallerIPC.exe' }}
```

Key points:
- Windows step uses `node build.js build-csharp` (existing build.js behavior, output to `dist/ModInstallerIPC.exe` per D-01)
- Linux step calls `dotnet publish` directly with self-contained flags, output to `dist/linux-x64/` per D-02
- Linux step sets chmod +x before upload per D-03 (no postinstall script)
- Both runners are per D-04: `ubuntu-22.04` + `windows-latest`
- The upload path for Windows captures only `dist/ModInstallerIPC.exe` (the single EXE); for Linux captures the `dist/linux-x64/` directory

**New `package-ipc` job (assembles tarball):**

```yaml
package-ipc:
  needs: build-ipc
  runs-on: ubuntu-latest

  steps:
    - uses: actions/checkout@v6

    - name: Install Node.js via Volta
      uses: volta-cli/action@v4

    - name: Setup pnpm
      uses: pnpm/action-setup@v4

    - name: Install dependencies
      run: pnpm install

    - name: Build webpack bundle
      run: pnpm --filter @nexusmods/fomod-installer-ipc build-webpack

    - name: Download Windows IPC binary
      uses: actions/download-artifact@v4
      with:
        name: ipc-win32-x64
        path: src/ModInstaller.IPC.TypeScript/dist/

    - name: Download Linux IPC binary
      uses: actions/download-artifact@v4
      with:
        name: ipc-linux-x64
        path: src/ModInstaller.IPC.TypeScript/dist/linux-x64/

    - name: Verify dist contents
      working-directory: src/ModInstaller.IPC.TypeScript
      shell: bash
      run: |
        echo "=== dist contents ==="
        find dist/ -type f | head -30
        echo "=== verify Windows EXE ==="
        test -f dist/ModInstallerIPC.exe && echo "OK: dist/ModInstallerIPC.exe exists"
        echo "=== verify Linux ELF ==="
        test -f dist/linux-x64/ModInstallerIPC && echo "OK: dist/linux-x64/ModInstallerIPC exists"
        test -x dist/linux-x64/ModInstallerIPC && echo "OK: dist/linux-x64/ModInstallerIPC is executable"

    - name: Pack tarball
      working-directory: src/ModInstaller.IPC.TypeScript
      run: npm pack

    - name: Upload package tarball
      uses: actions/upload-artifact@v4
      with:
        name: ipc-package
        path: src/ModInstaller.IPC.TypeScript/*.tgz
```

Key points:
- `package-ipc` depends on `build-ipc` (needs clause)
- Downloads both platform artifacts into the correct dist/ layout
- Runs webpack (platform-independent TypeScript bundling) only once
- Verifies both binaries exist and Linux binary is executable before packing
- `npm pack` captures everything in `dist/` per package.json `"files": ["dist/"]`

**IMPORTANT:** The `build-webpack` script is a separate build.js subcommand that only bundles TypeScript. The `pnpm --filter @nexusmods/fomod-installer-ipc build-webpack` invocation should work. If the package.json only has a `build` script (not `build-webpack`), use `node build.js build-webpack` in the working-directory instead. Check the `src/ModInstaller.IPC.TypeScript/package.json` scripts to confirm the correct invocation.
