**Replace `findStuckProcesses()`:**

```typescript
export async function findStuckProcesses(): Promise<number[]> {
  if (process.platform !== 'win32') {
    // Linux: pgrep -f matches against full command line (avoids 15-char /proc/comm truncation)
    try {
      const { stdout } = await exec('pgrep -f ModInstallerIPC');
      return stdout.trim().split('\n')
        .filter(line => line.trim() !== '')
        .map(line => parseInt(line.trim(), 10))
        .filter(pid => !isNaN(pid));
    } catch {
      // pgrep exits with code 1 when no processes found — not an error
      return [];
    }
  }

  // Windows: existing tasklist logic (unchanged)
  try {
    const { stdout } = await exec('tasklist /FI "IMAGENAME eq ModInstallerIPC.exe" /FO CSV');
    const lines = stdout.trim().split('\n');

    if (lines.length <= 1 || lines[0].includes('No tasks')) {
      console.log('No ModInstallerIPC.exe processes found.');
      return [];
    }

    const processes = [];
    for (let i = 1; i < lines.length; i++) {
      const columns = lines[i].split('","');
      if (columns.length >= 2) {
        const pid = columns[1].replace(/"/g, '');
        processes.push(parseInt(pid));
      }
    }

    return processes;
  } catch (error) {
    console.error('Error finding processes:', error.message);
    return [];
  }
}
```

**Replace `killProcess()`:**

```typescript
export async function killProcess(pid: number): Promise<boolean> {
  if (process.platform !== 'win32') {
    // Linux: SIGKILL ensures termination of stuck processes
    try {
      await exec(`kill -9 ${pid}`);
      console.log(`Successfully killed process ${pid}`);
      return true;
    } catch (error) {
      console.error(`Failed to kill process ${pid}:`, error.message);
      return false;
    }
  }

  // Windows: existing taskkill logic (unchanged)
  try {
    await exec(`taskkill /F /PID ${pid}`);
    console.log(`Successfully killed process ${pid}`);
    return true;
  } catch (error) {
    console.error(`Failed to kill process ${pid}:`, error.message);
    return false;
  }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TypeScript process spawner to child process | `RegularProcessLauncher.launch()` spawns an executable from a resolved path |
| cleanup-processes.ts to OS process table | `pgrep`/`kill` interact with system process table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | T (Tampering) | findExecutable() binary path | mitigate | Path resolved from `__dirname` (module's own directory) with `path.resolve()` normalization; no user-controlled input in path construction; `fileExists()` check before spawn |
| T-02-05 | E (Elevation of Privilege) | RegularProcessLauncher.spawn() | accept | Process launched with same privileges as parent Node.js process; no setuid/setgid; RegularProcessLauncher is explicitly the "no security restrictions" launcher |

| T-02-07 | S (Spoofing) | pgrep process name matching | accept | `pgrep -f ModInstallerIPC` matches command line substring; a malicious process named "ModInstallerIPC" would be killed, but this is a developer-facing cleanup utility, not a production security boundary |
</threat_model>
