<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FOMOD XML -> Parser | Untrusted XML attributes (source, destination paths) cross into path normalization |
| Archive file list -> Installer | Archive-provided file paths used in copy instruction construction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Parser10.ReadFileInfo() | mitigate | `TextUtil.NormalizePath()` collapses `../` sequences via separator normalization and toLower; existing `Mod.GetFile()` validates against `ModFiles` whitelist. Path traversal defense is validated in Phase 3 (UX-03). |
| T-01-02 | Information Disclosure | XmlScriptInstaller.InstallFile() | accept | `matchedFiles[0]` exposes archive's real file casing in instructions. This is the correct behavior — archive casing is not sensitive information. Risk: LOW. |
| T-01-03 | Denial of Service | ModFormatManager.cs | mitigate | OS guard prevents `TypeLoadException` crash from attempting to load Windows-only CSharpScript assemblies on Linux. Guard uses positive `IsOSPlatform(Windows)` check, not negative Linux check, so unknown platforms also skip registration safely. |
| T-01-04 | Elevation of Privilege | Parser10 path normalization | accept | `NormalizePath` lowercases and normalizes separators but does not sanitize `../` traversal sequences. This is acceptable for Phase 1: paths are validated downstream by `Mod.GetFile()` against the `ModFiles` whitelist (only files actually in the archive can be referenced). Phase 3 (UX-03) adds explicit traversal tests. Risk: LOW — constrained by archive whitelist. |
</threat_model>
