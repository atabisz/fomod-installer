# Quick Task 260409-k57 Summary

**Task:** Fix Phase 1 verification gap: emit UnsupportedFunctionalityWarning in Installer.cs when GetScriptType returns null but ScriptFilePath is non-null (C# script present on non-Windows)

## What was done

Added a guard block in `src/ModInstaller.Adaptor.Dynamic/Installer.cs` inside the `Install` method. After `progressDelegate(50)` and before the `HasInstallScript` branch, when `ScriptType == null`, the archive file list is scanned for a file matching `fomod/script.cs` (case-insensitive). If found, `Instruction.UnsupportedFunctionalityWarning("CSharpScript")` is added to the instruction list before falling through to `BasicModInstall`.

## Key design

- Guard is `ScriptType == null`, not an OS platform check — this correctly targets the condition where CSharpScriptType was not registered regardless of reason
- On Windows with `USE_CSHARP_SCRIPT`, `ScriptType` is non-null so the warning is never emitted; Windows behavior unchanged
- Fix committed to `linux-port` branch first, then merged to `master` per CLAUDE.md branch strategy
