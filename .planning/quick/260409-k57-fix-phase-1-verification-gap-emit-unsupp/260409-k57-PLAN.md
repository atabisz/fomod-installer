  <action>
In `Installer.cs`, in the `Install` method, add logic between line 101 (`await modToInstall.Initialize(validate)`)
and line 105 (`if (modToInstall.HasInstallScript)`) to detect and warn about unsupported C# scripts.

The detection logic:

After `modToInstall.Initialize(validate)` completes and before the `if (modToInstall.HasInstallScript)` check,
add a check: if `ScriptType` is `null` (no script type was resolved), scan `modArchiveFileList` for the
presence of a file matching the CSharpScript pattern -- i.e., a file whose name equals "script.cs"
(case-insensitive) inside a directory named "fomod" (case-insensitive). This mirrors the `scriptMatch` logic
in `ModFormatManager.cs` line 113-117.

Specifically, add this block after line 103 (`progressDelegate(50);`) and before line 105
(`if (modToInstall.HasInstallScript)`):

```csharp
// If no script type was resolved, check whether the archive contains a C# script
// file that we couldn't handle (e.g., CSharpScript not registered on non-Windows).
// Emit a warning so the caller knows functionality was skipped.
if (ScriptType == null)
{
    bool hasCSharpScript = modArchiveFileList.Any(f =>
        Path.GetFileName(f).Equals("script.cs", StringComparison.OrdinalIgnoreCase)
        && Path.GetFileName(Path.GetDirectoryName(f))
            .Contains("fomod", StringComparison.OrdinalIgnoreCase));
    if (hasCSharpScript)
    {
        Instructions.Add(Instruction.UnsupportedFunctionalityWarning("CSharpScript"));
    }
}
```

Why `ScriptType == null` and not an OS check: This is the correct guard because:
- On Windows with `USE_CSHARP_SCRIPT`, `CSharpScriptType` IS registered, `GetScriptType` returns
  a non-null `CSharpScriptType`, `HasInstallScript` is true, and the scripted path runs. The
  `ScriptType == null` check is false -- no warning emitted. Windows behavior unchanged.
- On Linux (or any build without `USE_CSHARP_SCRIPT`), `CSharpScriptType` is NOT registered,
  `GetScriptType` returns null, `ScriptType == null` is true, and if the archive has `script.cs`
  we emit the warning.
- On Windows WITHOUT `USE_CSHARP_SCRIPT` (edge case), the same correct behavior occurs: if the
  type isn't registered, the warning fires. This is correct.

The warning instruction is added to `Instructions` BEFORE the `if (modToInstall.HasInstallScript)`
block, so it will appear in the output alongside whatever BasicModInstall produces. This way the
caller (Vortex) sees the warning AND the basic file copies, which is the most useful behavior --
the user gets their files installed (best-effort) plus a notification that scripted installation
was not available.

Do NOT add `using System.Runtime.InteropServices` -- this fix does not use OS platform checks.
The existing `using System.Linq` (line 3) provides the `.Any()` method needed.
